# Built-in components

Every MDX page in LitMDX has access to a set of components without any import. They are injected globally via MDX's `components` prop and available in any `.mdx` file the moment you create it.

<Callout type="tip">
Need to add your own project-specific components or override a built-in? See [Custom Components](/features/customization/custom-components).
</Callout>

---

## Opt-in components

Some components ship with heavy third-party dependencies. Rather than bundling them unconditionally, LitMDX makes them **opt-in** — excluded from the default bundle unless you explicitly enable them in `litmdx.config.ts`.

This keeps the default JS payload minimal for the majority of documentation sites.

| Component | Config key | ~Size (gzip) | When to enable |
|---|---|---|---|
| [`<Mermaid>`](#mermaid) | `components.mermaid` | ~500 KB | Your docs contain diagrams |

To enable an opt-in component:

```ts
// litmdx.config.ts
import { defineConfig } from '@litmdx/core';

export default defineConfig({
  components: {
    mermaid: true,
  },
});
```

All other components listed on this page are always included — they have no external dependencies and add negligible bundle weight.

## `<Callout>`

A highlighted callout box. Use it to surface important notes, actionable tips, caveats, and breaking-change warnings alongside your content.

### Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `type` | `'note' \| 'tip' \| 'warning' \| 'danger'` | `'note'` | Visual variant and default title |
| `title` | `string` | type name | Override the default title |
| `children` | `ReactNode` | — | Callout body. Supports full Markdown. |

### Examples

```mdx
<Callout>
The `docsDir` option defaults to `docs/` relative to your project root.
</Callout>

<Callout type="tip">
You can nest code blocks, lists, and other Markdown inside any callout.
</Callout>

<Callout type="warning" title="Breaking change in v2.0">
The `sidebar_order` frontmatter key has been renamed to `sidebar_position`.
Update all your `.mdx` files before upgrading.
</Callout>

<Callout type="danger">
Never commit your `.env` file. Use environment variables injected at build time instead.
</Callout>
```

## `<Tabs>` and `<Tab>`

Tabbed content container. Each child `<Tab>` renders its content when its tab button is active. The active tab is tracked with local React state — no URL changes.

### `<Tab>` props

| Prop | Type | Description |
|---|---|---|
| `label` | `string` | Tab button label |
| `children` | `ReactNode` | Tab panel content |

### Example

````mdx
<Tabs>
<Tab label="npm">

Install using npm:

```bash
npm install litmdx
```

</Tab>
<Tab label="pnpm">

Install using pnpm (recommended for monorepos):

```bash
pnpm add litmdx
```

</Tab>
<Tab label="bun">

Install using Bun for the fastest installs:

```bash
bun add litmdx
```

</Tab>
</Tabs>
````

## `<Steps>`

Numbered step list. Wrap your content in `<Steps>` and use `### Heading` for each step — LitMDX automatically renders numbered circles using CSS counters, with no extra JavaScript.

### Props

| Prop | Type | Description |
|---|---|---|
| `children` | `ReactNode` | Content where `### Headings` become numbered steps |

### Example

````mdx
<Steps>

### Install LitMDX

```bash
bun add -D litmdx
```

### Create your docs directory

Create a `docs/index.mdx` file at your project root:

```mdx
---
title: Home
---

# Welcome
```

### Start the development server

```bash
bunx litmdx dev
```

Open [http://localhost:5173](http://localhost:5173) to see your docs site.

### Build for production

```bash
bunx litmdx build
```

The output is written to `dist/` — ready to deploy to any static host.

</Steps>
````

## `<Card>` and `<CardGrid>`

Navigation cards for index pages, feature overviews, and landing sections. A `<Card>` becomes a clickable link when `href` is provided.

### `<Card>` props

| Prop | Type | Default | Description |
|---|---|---|---|
| `title` | `string` | — | Card heading |
| `href` | `string` | — | Internal or external URL. Makes the whole card clickable. |
| `children` | `ReactNode` | — | Short description text |

### `<CardGrid>` props

| Prop | Type | Default | Description |
|---|---|---|---|
| `cols` | `2 \| 3 \| 4` | `2` | Number of columns |
| `children` | `ReactNode` | — | `<Card>` elements |

### Example

```mdx
<CardGrid cols={2}>
<Card title="Getting Started" href="/basics/getting-started">
Install LitMDX and publish your first docs site in under five minutes.
</Card>
<Card title="Configuration" href="/features/configuration">
Customize site title, navigation, base URL, and Open Graph metadata.
</Card>
<Card title="Components" href="/features/components">
Callout, Tabs, Steps, Card, Badge, CodeGroup — globally available in every MDX file.
</Card>
<Card title="Custom Components" href="/features/customization/custom-components">
Register your own React components and use them anywhere in your docs.
</Card>
</CardGrid>
```

## `<Badge>`

Inline label for API status, version numbers, feature flags, and similar metadata. Renders as a `<span>` — safe to use mid-sentence.

### Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `children` | `string` | — | Badge text |
| `variant` | `'default' \| 'success' \| 'warning' \| 'danger' \| 'outline'` | `'default'` | Color variant |

### Example

```mdx
## `createServer()` <Badge variant="success">Stable</Badge>

## `streamResponse()` <Badge variant="warning">Beta</Badge>

## `legacyConnect()` <Badge variant="danger">Deprecated</Badge>

Available since <Badge>v1.4.0</Badge>

Requires opt-in via <Badge variant="outline">Feature flag</Badge>
```

## `<CodeGroup>`

Groups multiple fenced code blocks under labeled tabs. The blocks and labels are matched by position — the first label corresponds to the first code block, and so on.

### Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `labels` | `string[]` | `['Code 1', 'Code 2', …]` | Tab label for each code block, in order |
| `children` | `ReactNode` | — | One fenced code block per tab |

### Example

````mdx
<CodeGroup labels={["npm", "pnpm", "bun", "yarn"]}>

```bash
npm install @litmdx/core
```

```bash
pnpm add @litmdx/core
```

```bash
bun add @litmdx/core
```

```bash
yarn add @litmdx/core
```

</CodeGroup>
````

## `<Mermaid>`

Renders [Mermaid](https://mermaid.js.org) diagrams directly in the browser. The diagram re-renders automatically when the user switches between light and dark mode.

<Callout type="warning" title="Opt-in required">
`<Mermaid>` is not included by default. It bundles mermaid, d3, dagre, and cytoscape — roughly 500 KB gzipped. Enable it only when your site actually uses diagrams:

```ts
// litmdx.config.ts
export default defineConfig({
  components: { mermaid: true },
});
```

When enabled, LitMDX consolidates all diagram dependencies into a single lazily-loaded `mermaid.[hash].js` chunk for efficient browser caching.
</Callout>

### Props

| Prop | Type | Description |
|---|---|---|
| `children` | `string` | Mermaid diagram definition |

### Example

````mdx
<Mermaid>
sequenceDiagram
  participant Browser
  participant S as LitMDX
  participant P as Pagefind

  Browser->>S: GET /docs/page
  S-->>Browser: HTML + JS bundle
  Browser->>P: search("query")
  P-->>Browser: results[]
</Mermaid>
````

<Mermaid>
sequenceDiagram
  participant Browser
  participant S as LitMDX
  participant P as Pagefind

  Browser->>S: GET /docs/page
  S-->>Browser: HTML + JS bundle
  Browser->>P: search("query")
  P-->>Browser: results[]
</Mermaid>

All [Mermaid diagram types](https://mermaid.js.org/intro/) are supported: flowcharts, sequence diagrams, class diagrams, ER diagrams, Gantt charts, and more.

<Callout type="note">
Mermaid diagrams render client-side and are not indexed by Pagefind search. A skeleton placeholder is shown while the diagram loads.
</Callout>
